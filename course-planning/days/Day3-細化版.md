## 🎯 詳細教學設計

### 0-5' 安全紅線儀式 + Day 1-2 回顧

**目的：** 
- 完成「三重安全儀式」（Day 1 → Day 2 → Day 3 的重申）
- 心理上開啟「從知道到做」的跨越

**講師開場稿（1 分鐘）：**

> 「各位，歡迎來到第三天。今天是最特別的一天——因為你們會看到自己的工作方法真的在 AI 裡活起來。
> 
> 但在開始之前，我想再提醒一次安全邊界。（投影 Day 1 三條紅線）
> 
> 如果說 Day 1 我們討論的是『AI 是什麼』，Day 2 我們討論的是『怎麼選工具』，那 Day 3 就是『怎麼用好工具，但不被工具綁架』。
> 
> 特別是，當你把自己的工作方法標準化成 Skill、交給 AI 去自動執行的時候，你需要定期檢查它的輸出——而不是盲目信任。
> 
> 所以今天的核心承諾是：**你可以協作，但你永遠是決策者。**」

**Day 1-2 關鍵詞快速回顧（2 分鐘）：**

投影一張總結卡：
```
Day 1：AI 不是聊天工具 → 是新型工作系統
       (LLM 是大腦，Tools 是雙手，Workflow 是肌肉記憶)

Day 2：工具選擇不用猜測
       三問框架：① 探索 vs 生產？② 需要最新資訊？③ 長流程多步驟？

Day 3：把工作方法交給 AI
       七問框架 → Skill 定義 → 90 天持續迭代
```

**學員視角檢查單（1 分鐘發放）：**

在學員手冊中放入一頁「我準備好了嗎？」：
```
□ 我理解 AI 的三層模型
□ 我能用三問框架選擇工具組合
□ 我有自己要優化的工作流
□ 我願意在 90 天內持續調整
□ 我知道 AI 輸出需要我的檢查

全勾的同學，今天會特別有收穫 😊
```

### 5-35' 七問 Skill 萃取框架工作坊

**核心邏輯：**

Skill = 把「我重複做的工作」標準化成「AI 能執行的流程」

七問不是理論，而是你和 AI 之間的「協議」。

**框架介紹（3 分鐘）：**

投影七問清單，逐問講解邏輯：

```
七問框架 — 把工作方法變成 AI Skill

第 1-2 問：定義邊界
   Q1: 這個工作叫什麼名字？誰會用它？在什麼情景下？
   Q2: 成功的標準是什麼？怎麼知道 AI 做好了？

第 3-4 問：拆解流程
   Q3: 第一步是什麼？需要什麼資訊作為輸入？
   Q4: 中間會有幾個關鍵步驟？每步的邏輯是什麼？

第 5-6 問：設定檢查點
   Q5: AI 最容易出錯的地方是哪裡？
   Q6: 你需要親自檢查哪些環節？（檢查點 = 你保留的決策權）

第 7 問：版本管理
   Q7: 用了三個月後，你想怎麼改進這個 Skill？
        (v1 → v2 → v3 的迭代路徑)
```

**案例示範 1：會議紀錄 → 行動項目清單（8 分鐘）**

**情景：** 市場部每週開會，產出 30 頁會議紀錄，需要人工整理成行動清單。

**講師現場示範七問回答：**

| 問題 | 答案 | 講師說明 |
|------|------|---------|
| Q1 | 「會議智能助理」；市場部經理；每周一次產品規劃會議 | 清楚的角色定位，AI 才知道說話的『人設』 |
| Q2 | 產出：3-5 項主要行動項目、責任人、deadline；用部長檢查 | 「成功」很具體——沒有模糊的 |
| Q3 | 輸入：會議原始紀錄（文字版或錄音逐字稿）；背景：上個月的行動清單 | AI 需要知道「我在哪裡」 |
| Q4 | 5 個步驟：① 逐句讀會議紀錄 ② 識別決策點 ③ 提取 Action items ④ 對標前月 ⑤ 格式化輸出 | 步驟數不超過 7，太多就不該自動化 |
| Q5 | 最容易混淆「討論中的想法」和「已決策的行動」| 語義判斷錯誤 = 最大風險 |
| Q6 | 我會檢查：行動項目的優先順序排序；deadline 的合理性；責任人是否同意 | 人的決策點清楚明確 |
| Q7 | v1（基礎版）→ v2（加入前月對標）→ v3（自動標記延期項） | 三個月的進化路線 |

**輸出：一張執行卡**

```
[Skill Card] 會議智能助理 v1

輸入：會議逐字稿 + 上月行動清單
步驟：
  1. 讀取會議記錄，標記所有「決策時刻」
  2. 提取句子，判斷是否為行動項
  3. 列出 3-5 項主要行動項，附責任人名字
  4. 與上月行動清單對照，標記『新增』『進行中』『完成』
  5. 按優先順序排列，輸出表格格式

檢查點（我來做）：
  ✓ 優先順序排序（依重要性 vs 時間）
  ✓ 責任人確認（只分配給有能力的人）
  ✓ Deadline 合理性（不要承諾不可能的時間）

下次迭代：加入『自動提醒延期項』的功能
```

**案例示範 2：顧客滿意度訪談 → 改進建議（10 分鐘）**

**情景：** 電商團隊每月做 50 份客戶訪談，整理成「客訴聚類」和「產品改進建議」。目前需要一周人工分析。

**講師現場示範七問回答：**

| 問題 | 答案 | 
|------|------|
| Q1 | 「客訴分析師」；產品經理和市場部；月度用戶體驗優化會議 |
| Q2 | 成功 = 產出 3-5 個客訴聚類 + 對應改進建議；產品經理簽核 |
| Q3 | 輸入：50 份訪談記錄（文字或錄音逐字稿）；背景：上月的聚類結果 |
| Q4 | ① 讀訪談，標記客訴語句 ② 分類（UI/功能/服務/價格）③ 數據聚類（相似客訴歸類）④ 生成改進假說 ⑤ 優先級排序 |
| Q5 | 最容易錯：把「個別吐槽」當成「系統性問題」；把「功能要求」當成「缺陷」 |
| Q6 | 我檢查：聚類的邏輯是否反映真實訴求（有時 5 個人的不同吐槽代表同一個問題）；改進建議是否可行 |
| Q7 | v1（基礎聚類）→ v2（加入訪談時間序列，找趨勢）→ v3（自動標記『緊急客訴』） |

**案例示範 3：新產品上線 Checklist（9 分鐘）**

**情景：** 創業團隊每季上線新產品，需要記錄 100+ 項檢查任務（測試、營銷、客服、法務）。目前靠 Excel，每次都漏項。

**講師現場示範七問回答：**

| 問題 | 答案 | 
|------|------|
| Q1 | 「產品上線檢查助手」；全部門經理；新產品發布前 2 周 |
| Q2 | 成功 = 產出完整 checklist，依部門分類，標記『必做』『可選』『風險項』；CEO 簽核 |
| Q3 | 輸入：產品功能描述（新增了什麼）；背景：上次上線的 checklist |
| Q4 | ① 分析新功能涉及的範圍 ② 推薦必做檢查項（過往踩過的坑）③ 按部門分類 ④ 時間排序 ⑤ 標記風險級別 |
| Q5 | 最容易錯：遺漏「冷門但高風險」的檢查項（比如舊系統相容性）；過度保守，導致發布延期 |
| Q6 | 我檢查：新功能的邊界情況（AI 不會自動想到）；時間估算（AI 傾向樂觀） |
| Q7 | v1（功能維度）→ v2（加入市場狀況）→ v3（自動學習『上次上線哪些項延誤了』） |

**小組練習（5 分鐘）：**

> 「現在輪到你們了。拿出你的工作流卡（Day 2 設計的那張）。我給你 5 分鐘，用七問框架，把這個工作流轉成一張 Skill 卡。
> 
> 不需要完美——重點是：你能清楚說出『AI 需要什麼輸入、怎麼處理、我檢查什麼』。」

**學員活動：**

在工作冊中發放「七問工作表」，結構如下：

```
我的 Skill 名稱：_________________________

Q1 定義：這是什麼工作？誰用？什麼場景？


Q2 成功標準：怎麼知道做好了？


Q3 輸入：需要什麼資訊？


Q4 流程：幾個主要步驟？


Q5 風險：最容易出錯的地方？


Q6 檢查點：我需要親自檢查什麼？


Q7 迭代：三個月後如何改進？
```

**講師巡場：** 走到每一組，問一個問題確認理解
- 「你的檢查點是什麼？」（幫助學員區分『AI 做的』vs『人做的』）
- 「如果 AI 弄反了，你怎麼發現？」（風險意識）

### 35-80' 現場跑卡儀式 ⭐ 這是課程的簽名時刻

**設定：** 投影屏幕上直播執行，全班圍觀，現場看到「卡片活起來」。

**目的：**
- 從「我知道 AI 怎麼用」→ 「我看到 AI 真的在用」
- 製造「哇」的情感時刻（不是冷冰冰的演示，是見證自己的工作被 AI 重新詮釋）
- 展示「版本迭代」的實際樣貌（v1 有缺陷 → 調整 prompt → v2 更好 → automation 可行）

**工作流（共 45 分鐘）：**

#### 第 1 張卡片執行（15 分鐘）

**選一張「簡單且通用」的卡片**

比如：「會議紀錄 → 行動清單」（從案例示範拿現成的，或請一位學員舉手分享）

**步驟：**

1. **讀卡（1 分鐘）**
   
   投影卡片，講師朗讀：「這是 XXX 同學的卡片。今天他要把會議紀錄變成行動項。我們看看 AI 怎麼做的。」

2. **實時執行 v1（Prompt 簡單版）（3 分鐘）**
   
   **Prompt 示範（講師在屏幕上輸入）：**
   ```
   你是會議助理。讀這段會議紀錄，列出 3-5 項主要行動項，
   包括：行動內容、責任人、deadline。
   
   會議紀錄：
   [貼入 1 段真實會議記錄，60-100 字]
   ```
   
   **AI 回應：** 投影 Claude/ChatGPT 的輸出
   
   全班看著結果生成。
   
   **講師評論：**
   > 「看，AI 抓出了 4 項行動項。但你們注意到了嗎？這裡有問題。」
   > （指向某個 action item）
   > 「它把『討論中的想法』當成了『已決策的行動』。這是 Q5『最容易出錯的地方』。」

3. **現場優化 v2（Prompt 加強版）（4 分鐘）**
   
   講師當著全班修改 Prompt：
   ```
   你是會議助理。讀這段會議紀錄，列出 3-5 項『已決策』的行動項。
   
   重要：只列出會議上『明確決定要做』的項目，不包括『正在討論』或『未來可能』的想法。
   
   每項包括：
   - 行動內容（清楚、可執行）
   - 責任人（誰確認了要做這件事？）
   - 最晚完成日期（會議上提到的日期，或推薦日期）
   - 優先級（高/中/低）
   
   會議紀錄：
   [同一段記錄]
   ```
   
   **AI 新回應：** 重新執行，結果改善
   
   **全班看到了什麼？** Prompt 調整 → 結果變好。這就是「版本迭代」。

4. **展示 v3（自動化版）（2 分鐘）**
   
   講師快速展示：
   > 「如果我們把這個 Prompt 固定下來，加上『自動檢查』的邏輯，會變成什麼樣？」
   
   投影 Zapier/Make 的自動化流程圖：
   ```
   Slack 新會議紀錄 → 
   Claude 自動執行 (v2 Prompt) → 
   檢查『優先級標記』 → 
   自動發送到 Notion 行動清單 → 
   周一自動提醒經理
   ```
   
   > 「這就是『Skill』。有了它，你就不用每次都手工跑一遍。AI 自動做。」

#### 第 2 張卡片執行（15 分鐘）

**選一張「中等複雜度」的卡片**

比如：「顧客反饋 → 改進建議」或「新產品上線 Checklist」

**重點轉向：讓學員參與「發現問題」**

1. **讀卡（1 分鐘）**

2. **第一次執行（3 分鐘）**
   
   簡單 Prompt 執行，故意展示「有問題」的版本。
   
   **講師邀請全班：**
   > 「各位，看這個結果。你們看出問題了嗎？」
   > 
   > （停頓 30 秒，讓學員指出）

3. **學員指出問題（2 分鐘）**
   
   「它混淆了什麼東西？」「遺漏了什麼？」
   
   講師確認：「對！這就是 Q5『最容易出錯的地方』。」

4. **現場優化（4 分鐘）**
   
   基於學員反饋，講師修改 Prompt，重新執行。
   
   全班看到改善。

5. **版本迭代討論（2 分鐘）**
   
   > 「三個月後，你覺得還想怎麼改進？」
   
   邀請卡片主人回答，討論 v2 → v3 的演進方向。

#### 第 3 張卡片執行（15 分鐘）

**選一張「最複雜」的卡片**

這次的重點：**展示『人 AI 協作』的關鍵節點**

1. **讀卡（1 分鐘）**

2. **多步驟執行（8 分鐘）**
   
   - 第 1 步：AI 生成初版
   - 第 2 步：人工檢查（邀請學員點評）
   - 第 3 步：AI 優化（根據反饋改）
   - 第 4 步：最終輸出
   
   全班看著「人 AI 協作」的過程實際進行。

3. **檢查點討論（4 分鐘）**
   
   > 「這個卡片，你的檢查點在哪裡？」
   > 「AI 的輸出，你最不放心的是什麼環節？」
   
   這鞏固了 Day 1-2 的「人永遠是決策者」的觀點。

4. **版本迭代（2 分鐘）**
   
   討論 90 天的改進路線。

**全班收穫：**

- 親眼看到 Prompt 調整 → 結果改善
- 親眼看到「檢查點」的實際樣子
- 親眼看到「版本迭代」怎麼進行
- **心理轉變：從「AI 很神奇」→ 「AI 是工具，我可以調教」**

**技術準備（講師課前檢查表）：**

- [ ] 確認 Claude/ChatGPT 頁面已開啟，登錄狀態正常
- [ ] 三張卡片的 Prompt 已預先測試，結果存好備份
- [ ] 投影屏幕明亮（遠處學員能看清）
- [ ] 鍵盤輸入速度適中，讓全班有時間跟上思路
- [ ] 有備用 Wi-Fi 熱點，以防主網絡故障

### 80-105' 90 天落地計畫設計工作坊

**核心邏輯：**

課程不是結束點，而是「啟動儀式」。

90 天是從「知道」到「變成習慣」的關鍵週期。

**框架介紹（3 分鐘）：**

投影 90 天分期表：

```
Days 1-30：建立基礎 (Habit Formation)
  選 3 個重複的工作流
  為每個寫一張工作流卡
  用 AI 執行 1 次

Days 31-60：持續調整 (Iteration & Testing)
  每週執行一次工作流
  記錄『失敗案例』
  修改 Prompt，改進結果

Days 61-90：標準化與分享 (Systemization & Leverage)
  1 個工作流達成『完全自動化』
  分享給團隊或同儕
  複製到其他工作流
```

**為什麼 90 天？** 

- 30 天：養成新習慣的最小週期
- 60 天：發現問題、反覆調整
- 90 天：習慣固化，可以教他人

**工作坊步驟（22 分鐘）：**

#### 步驟 1：回顧你的三個工作流（5 分鐘）

發放「工作流盤點表」：

```
今天上課，我學到的三個 AI 工作流：

① _________________________
   目前需要多久？ _____ 小時
   AI 可以節省多久？ _____ 小時
   優先級（高/中/低）：

② _________________________
   目前需要多久？ _____ 小時
   AI 可以節省多久？ _____ 小時
   優先級（高/中/低）：

③ _________________________
   目前需要多久？ _____ 小時
   AI 可以節省多久？ _____ 小時
   優先級（高/中/低）：
```

**講師指導：**

> 「選的標準很簡單：你一周內會重複做 2 次以上。如果一個月才做一次，先跳過，因為優先級不高。」

#### 步驟 2：Days 1-30 計畫（5 分鐘）

## 💡 講師 FAQ

**Q: 如果卡片執行時 AI 犯了一個大錯誤，怎麼辦？**

A: 不要馬上修復。停留在「有問題」的狀態，邀請全班一起診斷：「為什麼會出錯？怎麼調整 Prompt？」
這個過程比「完美執行」更有教育意義。

---

**Q: 學員說「我沒有工作流，只有單調的日常任務」。**

A: 讓他選「整理郵件」「做週報」「客戶回覆」等重複任務。
任何「一周做 2 次以上」的事，都可以自動化。

---

**Q: 如果現場網絡故障，無法執行 Prompt？**

A: 準備備用方案：
1. 錄好影片，現場播放
2. 用預先測試的截圖展示結果變化
3. 描述 Prompt 調整過程，邀請學員「想象」結果

---

**Q: 學員填完 90 天計畫後，然後呢？**

A: 7 天後主動跟進。
在社群發起「Week 1 check-in」，邀請所有人分享進度。
建立「social accountability」——有人陪著，堅持率會提高。

---

**Q: 怎麼確保學員真的會執行 Skill，而不是上完課就算了？**

A: 三個機制：
1. **社群挑戰**：有獎勵（點讚、徽章）
2. **診所時間**：卡住時有人幫
3. **同儕壓力**：看到別人分享進度，自己也會跟

心理學告訴我們，「公開承諾 + 社群見證」的堅持率最高。

## 🎯 Day 3 的核心設計邏輯

| 維度 | 做法 | 為什麼 |
|------|------|--------|
| **七問框架** | 不講理論，用案例「示範」七問怎麼用 | 讓學員看到「框架」是活生生的工具，不是空洞概念 |
| **現場跑卡** | 投影執行，邀請學員發現問題 | 「我看著 AI 出錯、看著你修復」的體驗，比聽你講故事印象深 100 倍 |
| **版本迭代** | v1（錯） → v2（改） → v3（自動化）的實時演進 | 學員明白：改進不是一蹴而就，是「反覆測試 → 發現問題 → 調整 → 再測」的循環 |
| **90 天計畫** | 承諾式、分期式、可追蹤 | 把抽象的「課後要用」變成具體的「Week X 要完成什麼」 |
| **社群延續** | 從「課程」轉到「持續學習社群」 | 課程是開始，不是結束。社群 = 複利效應的開端 |
| **結業儀式** | 複誦 + 簽署 + 社群銜接 | 心理學的「儀式感」強化承諾，也讓學員對「接下來要做什麼」有清晰認知 |
