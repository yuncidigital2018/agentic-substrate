# Day 3 Compare
# Day 3 比較表 - AI 生成靈魂融入日常

## Core Goal

Help learners turn an everyday work process into a reusable, testable AI workflow card.

讓學員把日常流程變成可重複使用、可測試的 AI 工作流雛形。

## Version Comparison

| Version | Core Design | Strength | Risk | Keep / Modify / Drop |
|---|---|---|---|---|
| Skill intro version | Explains Skill as reusable expertise | Strong connection to Agentic Substrate | May feel abstract | TBD |
| Workflow-card version | Learners complete seven-question card | Concrete and usable | Needs examples and pacing | TBD |
| Exchange-test version | Learners run each other's cards | Makes transferability visible | Time pressure | TBD |
| Live-run ending version | Instructor runs 2-3 learner cards live | Strong final memory | Needs careful selection | TBD |

## Candidate Final Elements

- Reverse design frame: "What does AI need to know to do this well?"
- Seven-question workflow card.
- Pair exchange and run test.
- Live execution of selected workflow cards.
- Version path:
  - `Workflow Card v1 -> Optimized Prompt -> Checklist -> Semi-Automated Workflow -> Skill / Agent`
- 90-day AI landing plan.

## Open Questions

- Should learners work on personal, team, or local-case workflows?
- Should the workflow card be in Notion, Google Doc, paper, or HTML?
- How many cards can be live-run in 15 minutes?
- What criteria should be used to select cards for live execution?
