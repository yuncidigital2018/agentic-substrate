## 三、【5-30 分鐘】AI Tool Picker：三問框架

### 核心邏輯

**講師話術**

> 「昨天我們看一個任務『30 人活動規劃』。用三個不同的工具組合，結果完全不同。
>
> 那如果換個任務呢？『整理 100 份客戶回饋，找出最常見的問題』。用什麼工具？
>
> 或『設計一場 5 天 4 夜的旅程』。又用什麼？
>
> **關鍵是，沒有『最強的工具』，只有『最適合這個任務的工具』。**
>
> 怎麼判斷？很簡單，三個問題。」

---

### 三問框架

#### **問題 ①：這個任務是『探索』還是『生產』？**

| 探索型任務 | 生產型任務 |
|----------|----------|
| 還沒有明確答案 | 已知道要產出什麼 |
| 需要信息收集、研究、比較 | 需要按既定邏輯執行 |
| 例：「該不該進入新市場？」「競爭對手怎麼做的？」 | 例：「寫邀請信」「整理會議紀錄」「設計問卷」 |
| **適合工具：ChatGPT、Gemini（搭配網頁搜尋）** | **適合工具：Claude、ChatGPT（有明確結構）** |

#### **問題 ②：需要『最新資訊』嗎？**

| 需要最新資訊 | 不需要最新資訊 |
|------------|------------|
| 例：「2024 台灣電商市場趨勢」「最新 AI 工具排名」「房價行情」 | 例：「寫文案模板」「分析過去案例」「結構化想法」 |
| **適合工具：ChatGPT（網頁搜尋模式）、Gemini（有搜尋能力）** | **適合工具：Claude、ChatGPT（訓練資料足夠）** |

#### **問題 ③：需要『長流程、多步驟、多工具串接』嗎？**

| 複雜長流程 | 簡單單一任務 |
|----------|-----------|
| 例：「市場研究 → 財務模型 → 簡報 → 報告」 | 例：「改文件格式」「寫一封信」 |
| **適合工具：Agent 工作流 + 多個 AI 工具** | **適合工具：單一 AI（Claude 或 ChatGPT）** |

---

### 決策樹（投影）

```
你的任務來了
     │
     ├─ 是『探索』嗎？
     │  ├─ YES → 需要『最新資訊』嗎？
     │  │         ├─ YES → ChatGPT（網頁搜尋）或 Gemini ⭐️
     │  │         └─ NO → Claude / ChatGPT
     │  │
     │  └─ NO（是『生產』）→ 複雜流程嗎？
     │                      ├─ YES → Claude + Gemini Canvas
     │                      └─ NO → Claude / ChatGPT
```

---

### 工具選擇決策卡（學員手冊版本）

```
我的任務：_______________________

① 這是探索還是生產？
   □ 探索（還在找答案）
   □ 生產（知道要做什麼）

② 需要最新資訊嗎？
   □ 是（市場、新聞、價格...）
   □ 否（通用知識、分析...）

③ 多步驟、複雜流程嗎？
   □ 是（需要多個工具串接）
   □ 否（單一任務）

👉 建議工具組合：_________________
   第一步用________，第二步用________
```

### 講師話術（帶過框架）

> 「很簡單，對吧？不用記所有工具名字。只要問自己這三個問題，答案就出來了。
>
> 明年新工具更多，但這三個問題永遠有效。
>
> **工具會變，決策邏輯不變。**」

## 四、【30-60 分鐘】工具組合案例：3 個精選

每個案例用「**任務 → 三問判斷 → 工具組合 → 執行流程**」的格式呈現。

**執行方式：** 講師現場示範 1-2 個重點案例，或用預錄短影片（各 2-3 分鐘）。重點是讓學員**看到『工具組合是怎麼串接的』**，而不是講工具本身。

---

### **案例 1：會議紀錄 → 行動項目 → 跟進信**

**情景：**
> 你是專案經理。開完一場 1 小時的團隊會議，有逐字稿或錄音。需要產出：
> - 決議事項
> - 待辦項目（誰、何時完成）
> - 寄給團隊的跟進信

**三問判斷：**
```
① 探索或生產？ → 生產（會議已開完，要整理）
② 需要最新資訊？ → 不需要（資訊都在逐字稿裡）
③ 複雜流程？ → 是（逐字稿 → 結構化 → 信件 → 發送）
```

**工具組合：**
```
1️⃣ Claude / ChatGPT
   上傳逐字稿或錄音記錄，問：
   「幫我從以下會議紀錄中擷取：
   - 三項主要決議
   - 五個待辦項目（含負責人、截止日期）
   
   用表格格式呈現。」

2️⃣ Claude
   根據上面的表格，寫一份信：
   「根據會議摘要，寫一份給團隊的信。
   內容包括：感謝、主要決議、待辦事項提醒、下次會議時間。
   300 字以內，語氣親切但專業。」

3️⃣ Gmail / Outlook → 複製貼上，發信
```

**演示視覺化：**
```
Claude（擷取） → Claude（寫信） → 你（發信）

產出：
┌────────────────────────────────┐
│ 會議決議表                      │
├──────────┬──────────┬──────────┤
│ 項目     │ 負責人   │ 截止日期 │
├──────────┼──────────┼──────────┤
│API 文件  │ 王小明   │ 5/20     │
│QA 測試   │ 李小華   │ 5/23     │
└──────────┴──────────┴──────────┘

跟進信開頭：
感謝各位昨天的投入。
本次會議的三項主要決議是：
1. 下週五前完成 API 文件
2. QA 測試範圍涵蓋行動版
3. 預留成本調整 15%

待辦事項詳見上表...
```

**時間成本：** 20 分鐘完成（傳統方式要 45 分鐘-1 小時）

### **案例 2：🌾 5 天 4 夜旅程規劃 + 報名表單設計**

**情景：**
> 你在地方創生組織工作。要為雲林小農設計「5 天 4 夜的農村沉浸體驗」，目標是吸引都市上班族。需要：
> - 完整行程規劃（含農場參訪、DIY、文化體驗、住宿）
> - 報名表單的欄位設計（要收集什麼資訊？）
> - 宣傳文案

**三問判斷：**
```
① 探索或生產？ → 兼具
   - 探索部分：這個農村有什麼特色？怎麼設計體驗？
   - 生產部分：設計完整行程、設計表單欄位
   
② 需要最新資訊？ → 部分是（農產品季節、交通、住宿）

③ 複雜流程？ → 是（查資料 → 設計行程 → 設計表單 → 寫文案）
```

**工具組合：**
```
1️⃣ Claude
   「我要設計『5 天 4 夜雲林農村沉浸體驗』行程。
   目標客群：30-45 歲上班族，想逃離壓力。
   你知道雲林農村體驗的常見項目嗎？
   包含：農場參訪、DIY 活動、在地美食、住宿、交通。
   用時間表呈現。」

2️⃣ Claude
   「根據上面的行程，設計『報名表單』的欄位。
   需要收集什麼資訊？
   建議欄位包括：基本資訊、動機、飲食限制、期望等。
   用 CSV 格式列出。」

3️⃣ Claude 或 Gemini Canvas
   「根據行程和表單欄位，製作一個 HTML 表單或簡單的互動式表單。」

4️⃣ Claude
   「根據行程，寫一份 300 字的活動介紹文案。
   開頭要戳中上班族的痛點（工作壓力、疲憊、想連結...）
   然後描述這個活動怎麼解決這些痛點。」
```

**演示視覺化：**
```
Claude（設計行程 + 表單欄位 + 文案）
    ↓
Gemini Canvas（建互動式表單）
    ↓
發布到網站或分享連結

產出行程示例：
Day 1 | 14:00 集合 → 15:00 稻田導覽 → 17:00 農民開講 → 18:30 農家晚餐
Day 2 | 06:30 日出觀察 → 08:00 自製面膜 DIY → 12:00 農家便當 → 14:00 製作豆腐
Day 3 | ...

報名表單欄位：
- 姓名 / email / 電話
- 來自哪個城市
- 為什麼想來？（複選：逃離壓力、親近自然、學習技能、其他）
- 飲食限制
- 期望在這裡得到什麼？
```

**時間成本：** 1.5-2 小時完成（傳統方式要 4-5 小時）

**[重要]** 這個案例特別適合地方創生、社區營造的學員。講師要強調：
> 「同一套邏輯，用在農村、用在社區活動、用在青年提案，都一樣。決策框架不會變。」

### **案例 3：客戶回饋問卷設計 + AI 聚類分析**

**情景：**
> 你是產品經理。需要收集客戶對產品的回饋，並分析最常見的問題。但問題是：**『怎麼設計一份『能收集有效回饋』的問卷？』**

**三問判斷：**
```
① 探索或生產？ → 生產（知道要做什麼：設計問卷、收集回饋、分析）

② 需要最新資訊？ → 不需要（回饋會陸續進來）

③ 複雜流程？ → 是（問卷設計 → 收集回饋 → 分析聚類 → 優先順序）
```

**工具組合：**
```
1️⃣ Claude
   「我要設計一份『客戶產品回饋問卷』。
   產品特點：[你的產品介紹]
   目標：找出『最常見的 3 個痛點』和『改善建議』
   
   請幫我設計問卷：
   - 用開放式題目（避免誘導客戶說特定答案）
   - 包含『為什麼』（找根本原因，不只表面）
   - 長度不超過 10 題（提高填寫率）
   
   列出問題和選項。」

2️⃣ Claude
   「優化問卷措辭，避免『誘導性提問』和『業界術語』。
   讓各種教育程度的客戶都能理解。」

3️⃣ Gemini Canvas
   「根據優化後的問卷，製作一個互動式問卷表單，可以分享給客戶填寫。」
   （或手動用 Google Form / Typeform 收集）

4️⃣ Claude
   收集到足夠回應後（例 30-50 份），把回應資料複製貼進來，問：
   「幫我分析這份客戶回饋問卷：
   1. 聚類成問題主題（例：介面難用、缺少功能、價格問題...）
   2. 統計每個問題出現的次數
   3. 列出前 3 個最常見的問題
   4. 針對每個問題，建議改善方案的想法
   
   用表格呈現。」

5️⃣ Claude
   「根據聚類結果，寫一份『改善優先順序』報告給 CEO。
   需要說明：為什麼優先做這 3 個、預期效果、所需資源。」
```

**演示視覺化：**
```
Claude（設計問卷）
    ↓
Gemini Canvas（製作互動表單）
    ↓
收集回應資料
    ↓
Claude（分析聚類）
    ↓
Claude（寫報告）

產出示例：

問題聚類表：
┌──────────────┬──────┬────────────┐
│ 問題主題     │ 次數 │ 代表回饋   │
├──────────────┼──────┼────────────┤
│ 介面太複雜   │ 18   │「不知道...│
│ 缺少 X 功能  │ 15   │「想要...  │
│ 價格太高     │ 12   │「比起...  │
└──────────────┴──────┴────────────┘

改善優先順序報告：
1️⃣ 簡化介面（影響 18 位客戶，技術難度中）
2️⃣ 加 X 功能（客戶呼聲高，技術難度高）
3️⃣ 調整定價（影響留存率，商業優先順序）
```

**時間成本：** 45 分鐘完成（傳統方式要 2-3 小時）

---

### 講師環節結尾

> 「看到了嗎？三個完全不同的任務，但決策邏輯一樣：
> 1. 問自己三個問題
> 2. 選對工具組合
> 3. 按流程執行
> 
> **工具永遠在變，但框架永遠有效。**
>
> 現在，輪到你設計你自己的工具組合。」

## 五、【60-100 分鐘】反向挑戰實作（40 分鐘充裕版）

### 核心目的

讓學員**先自己思考『不用 AI 怎麼做』**，然後「發現 AI 能幫的地方」。這樣學到的不只是「工具」，而是「思路」。

---

### 執行方式

**給每組學員一個任務卡（小組 3-4 人）**

每組選**一張任務卡**，依序回答三個問題，最後填寫《AI 工具組合卡》。

#### **任務卡 A：寫一份青年創業融資簡報**

```
背景：你的創業團隊要向投資者融 500 萬元。
     投資者會有 30 分鐘的會面時間。
     需要一份 10-15 頁的簡報。

三個問題（依序回答）：

① 「不用 AI，你會怎麼做？」
   （列出步驟：搜資料 → 整理想法 → 設計簡報 → 排版...）

② 「這個流程中，哪些環節『重複性高』或『容易卡住』？」
   （挑出 2-3 個可以用 AI 加速的地方）

③ 「如果用 AI，你會怎麼用？用什麼工具？」
   （設計 AI 工具組合）

最後，填寫《AI 工具組合卡》：
┌──────────────────────────┐
│ 步驟 1：研究融資內容      │
│ 工具：Perplexity（查案例）│
│                        │
│ 步驟 2：產出簡報大綱     │
│ 工具：Claude（結構化）   │
│                        │
│ 步驟 3：設計簡報        │
│ 工具：Gemini Canvas     │
│                        │
│ 步驟 4：最後覆核        │
│ 工具：你自己            │
└──────────────────────────┘
```

---

#### **任務卡 B：設計一個社區老人陪伴活動（地方創生場景）**

```
背景：你在社區發展協會。要設計「為獨居老人設計的陪伴活動」。
     需要：活動企劃、宣傳文案、與子女溝通的信件、報名系統。

三個問題：

① 「不用 AI，你會怎麼做？」

② 「AI 可以在哪些地方幫忙？」
   （提示：查在地資源、設計活動流程、寫文案、建表單）

③ 「設計你的工具組合。」
```

---

#### **任務卡 C：整理一份 50 份客戶意見的分析報告**

```
背景：你收到 50 份客戶對你的服務的意見。
     需要：找出最常見的 3 個問題，並提出改善方案。

三個問題：

① 「不用 AI，你會怎麼做？」
   （想像自己用 Excel、手寫 Post-it...）

② 「AI 可以在哪些地方幫忙？」
   （提示：分類、統計、分析、寫報告）

③ 「設計你的工具組合。」
```

---

### 執行節奏（40 分鐘）

- **0-3 分鐘**：講師解釋「反向挑戰」的意義
- **3-15 分鐘**：小組討論題目（安靜思考 + 互相討論）
- **15-30 分鐘**：小組填寫《AI 工具組合卡》（講師巡場，一對一指導）
- **30-38 分鐘**：快速分享（挑 2-3 組講他們的想法）
- **38-40 分鐘**：講師點評 + 過渡到 Day 3

### 講師話術（開場）

> 「現在開始反向挑戰。
>
> 什麼叫『反向』？我不先告訴你『用 AI 怎麼做』。
>
> 而是，我先問你『不用 AI 怎麼做？』你說出整個流程，然後我們一起看『AI 能在哪裡幫忙』。
>
> 為什麼要這樣做？因為**最大的浪費是『用 AI 做不值得做的事』**。
>
> 你得先理解『這個事值不值得優化』，才能決定用不用 AI。」

### 講師巡場重點

- ⚠️ 有沒有小組卡在「不用 AI 也不知道怎麼做」？提示他們「怎麼笨辦法都行，就是比較慢」
- ✅ 有沒有小組識別出「重複性最高」或「最浪費時間」的環節？
- 🎯 他們填的 AI 工具組合，合理嗎？有沒有過度設計？

### 講師點評重點（38-40 分鐘）

挑一個「工具組合設計得好」的小組，簡單讚賞：

> 「A 組想得很清楚。他們說『不用 AI 的方式是 Excel 手工聚類，很容易遺漏』，所以選了『Claude 分析 + Google Form 收集』。這個邏輯很對。」

## 六、【100-120 分鐘】Day 3 預告

### 講師話術

> 「好，現在你知道怎麼選工具了。
>
> 但是，有個新問題：**『這個工具組合，怎麼讓別人也能用？怎麼下次不用重新設計？』**
>
> 比如，你花了 1.5 小時設計了『客戶回饋 → 聚類 → 報告』的工具組合。
> 下個月，又收到 50 份客戶意見。
> 你是要重新設計一遍，還是有辦法『直接套用』？
>
> **答案是有的。這就是 Day 3 要學的『Skill』。**
>
> 把你的工具組合『文件化』『標準化』『可複用化』。
>
> 到時候，一個『工作流卡』在手，下次有類似任務，直接改資料，流程自動跑。
>
> **最重要的是，現場我們要示範：你的卡片被 AI 真的執行了。**
>
> 那是最震撼的時刻。」

### 核心記憶點複誦

> **「工具會變，決策邏輯不變。」**

## 八、可複製的 Prompt 庫

### Prompt 1：問卷設計幫手

```
我要設計一份客戶回饋問卷。

產品背景：[你的產品簡介]
目標：找出『最常見的 3 個痛點』

請幫我設計問卷：
- 用開放式題目（讓客戶自由表達，不誘導）

- 總長不超過 10 題（提高填寫率）
- 語言簡單（各教育程度都懂）

列出問題和說明。
```

### Prompt 2：行程規劃幫手

```
我要設計一份「5 天 4 夜的 [地點] 體驗行程」。

地點特色：[有什麼資源？農產品？文化？]
活動形式：[DIY、參訪、用餐、住宿...]

請幫我設計完整行程：
- 含時間表
- 每天的核心體驗
- 餐飲和住宿安排
- 交通方式

用時間線呈現。
```

### Prompt 3：回饋分析幫手

```
我有 [數量] 份客戶回饋。

3. 列出前 3 個最常見的問題
4. 針對每個問題，提供改善方案的想法

用表格呈現。

[貼上回饋資料或上傳檔案]
```
