## Cross-Session Reminders - 跨堂執行關鍵提醒

| Item | Recommended Approach |
|---|---|
| Audience | Clearly position the course for people with basic office/project experience who have not used AI systematically. Avoid making beginners feel lost or advanced users feel under-challenged. |
| Time Allocation | Two hours is tight. Use roughly 20% concepts, 50% live demos, 30% hands-on work plus peer/instructor feedback. Send handouts in advance so class time can stay active. |
| Environment Setup | Send an account and network checklist before class. Prefer free trials or unified test accounts where possible to avoid losing time to registration, payment, or firewall issues. |
| Risk Framing | Include hallucination verification and clear boundaries for confidential/personal data. Cover anonymization, redaction, and possible local/private deployment alternatives. Enterprise learners will care about this. |
| Continuity | Build a Notion/Airtable template library where learners can upload, fork, and iterate on their workflow cards. This can naturally extend into a paid community or consulting pipeline. |

## Day 1 Upgrade - 第一堂加一個震撼點

Day 1 is already strong as a cognitive upgrade, but it needs one "shock demo" that makes the shift visible.

Also add a short section:

**Where are AI's boundaries?**

Use this to manage expectations early:

- AI is not a truth machine.
- AI is not always cheaper than human work.
- AI is strongest when the task, context, criteria, and review loop are clear.
- AI becomes risky when the user cannot judge the output quality.

Use a 10-minute live theater:

**Same task**

> Help a small farmer in Yunlin design a 3-day, 2-night experience itinerary, promotional copy, and registration page content.

Run it three ways:

1. Traditional workflow: search, Word, manual copy/paste
2. Single ChatGPT conversation
3. Agent workflow: decomposed steps plus multiple tools

Focus the comparison on process cost:

- How many steps?
- Who has to create the structure?
- Can it be reused?
- Can it be checked?

Teaching point:

> "AI as computation layer" becomes real when learners see the process cost collapse.

If time is limited, simplify the five-layer model into three layers:

```text
LLM -> Tools / Agents -> Workflow
```

This avoids overload while preserving the core idea.

Add an "instant experience" segment:

- Give learners the same small task.
- Ask them to complete it in two or three different ways.
- Compare process, output, and effort.

## Day 2 Upgrade - 第二堂補一個決策框架

Day 2 needs a simple decision framework so learners stop asking, "Which tool should I use?"

Suggested name:

**AI Tool Picker**

Three questions:

1. Is this task exploration or production?
2. Does it need the latest information?
3. Does it require a long, multi-step workflow?

Mapping:

- Exploration + latest information: Perplexity / Gemini
- Deep writing or structure: Claude / ChatGPT
- Long workflow: Agent / automation tools
- Visual output: Canva / generative design tools

Teaching point:

> Learners should leave with the ability to choose tools without needing to ask someone else every time.

Add a "reverse challenge" exercise:

- Give learners one realistic task.
- First ask: "How would you do this without AI?"
- Then ask: "Which parts become easier with AI?"
- Finally ask: "Which parts still need human judgment?"

This helps learners see AI's real value instead of treating it as magic.

Add a "tool combination case library" with 3-5 verified workflow templates, such as:

- Research -> summary -> decision memo
- Meeting notes -> action items -> follow-up email
- Local activity idea -> copywriting -> registration page draft
- Personal profile -> one-page website -> social post
- Customer feedback -> issue clustering -> improvement plan

Add a "trap warning" section:

- Tasks that require confidential data without anonymization
- Tasks where the learner cannot verify correctness
- Tasks with high legal, medical, or financial risk
- Tasks where prompt setup costs more than doing the work
- Tasks requiring human trust, consent, or sensitive relationship handling

## Day 3 Upgrade - 第三堂讓成果真的能帶走

Day 3 is the strongest differentiator. Make the output immediately usable.

Upgrade 1: Make the "AI workflow card" executable.

The card should be usable as:

- A prompt for GPT / Claude
- A Notion / Google Doc template
- A prompt plus checklist
- A future SKILL.md skeleton

The seven-question extraction method should become a fillable card template completed during class.

Teaching point:

> Learners should be able to use what they wrote on the same day.

Upgrade 2: Add exchange and reuse.

Learners pair up:

1. Exchange workflow cards
2. Try to run the other person's workflow
3. Give feedback on missing inputs, unclear steps, and validation gaps

Teaching point:

> A good workflow is operable by someone else.

This creates the lived experience:

> Skill = transferable expertise.

Add a reverse design frame:

Do not only ask:

> How do I use AI to finish this work?

Also ask:

> What does AI need to know in order to do this work well?

This shifts learners from a user perspective to a designer perspective.

Suggested framing:

> When you want to delegate work to AI, you first need to answer seven questions clearly.
> These seven questions are not only for AI. They train you to make your tacit expertise explicit.
> If you cannot explain it clearly, it means you may not fully understand the work yet.

Add version iteration:

```text
Workflow Card v1 -> Optimized Prompt -> Repeatable Checklist -> Semi-Automated Workflow -> Skill / Agent
```

This helps learners understand that AI workflow design is not one-and-done. It improves through use.

Close with a concrete "90-day AI landing plan" instead of a general inspirational ending:

- Days 1-30: Pick 3 repeated tasks and write workflow cards.
- Days 31-60: Test them weekly and record failure cases.
- Days 61-90: Standardize 1 workflow and share it with a team or peer group.

> You are not just using AI. You are designing a system that works with you.

> 你不是在用 AI，你是在設計一個會幫你工作的系統。

Three-day variation:

- Day 1: AI is a system.
- Day 2: You are designing the system.
