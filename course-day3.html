Seven Questions

七問框架：把工作方法變成 AI Skill

定義邊界

Q1 這是什麼工作？誰用？什麼場景？
Q2 成功標準是什麼？

拆解流程

Q3 需要什麼輸入？
Q4 中間有幾個關鍵步驟？

設定檢查點

Q5 最容易出錯在哪？
Q6 人要檢查什麼？
Q7 三個月後怎麼改進？

七問不是給 AI 看的，而是訓練你把隱性經驗說清楚。
Skill Card Builder

我的 Skill 卡

填寫七問後，這裡會產生可丟給 AI 的 Skill 卡草稿。
Signature Moment

現場跑卡儀式：讓卡片活起來

第一張：簡單通用

會議紀錄 -> 行動清單。展示 v1 有缺陷，v2 如何透過 Prompt 調整改善。

第二張：中等複雜

顧客回饋 -> 改進建議。讓學員參與發現問題與優化。

第三張：人 AI 協作

展示多步驟執行、人工檢查點與最終輸出。

心理轉變：從「AI 很神奇」到「AI 是工具，我可以調教」。
Prompt Iteration

v1 -> v2：看見版本迭代

v1 簡單版

你是會議助理。讀這段會議紀錄，列出 3-5 項主要行動項，包括：行動內容、責任人、deadline。

v2 加強版

只列出會議上「明確決定要做」的項目，不包括正在討論或未來可能的想法。責任人沒有明確同意，標記為待確認。
重點不是 Prompt 寫很長，而是把 Q5 風險和 Q6 檢查點寫進去。
90 Days

90 天落地計畫：從課堂變成習慣

階段行動目的
Days 1-30選 3 個重複工作流，為每個寫一張卡，至少跑一次建立基礎
Days 31-60每週執行一次，記錄失敗案例，修改 Prompt v1 -> v2持續調整
Days 61-90標準化 1 個 Skill，分享給同事或同儕，收集回饋系統化與擴散
90 天之後，Skill 就不只是個人工具，而是團隊資產。
